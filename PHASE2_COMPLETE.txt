================================================================================
                    CACHE SYSTEM ARCHITECTURE
================================================================================

Cache Storage:
  Location: .cache/ollama_analysis/
  Database: SQLite3 (analysis_cache.db)
  Format: .db file (portable, reliable)

Cache Key Generation:
  Key = SHA256(template-id + host + model + analysis_type)
  Deterministic (same input = same key)
  Collision-resistant

Cached Data:
  - vulnerability_hash
  - vulnerability_data (JSON)
  - model name
  - analysis_type
  - LLM response
  - Timestamps
  - Hit count

Database Schema:
  cache (PRIMARY TABLE)
  ├── cache_key (PK)
  ├── vulnerability_hash (INDEX)
  ├── vulnerability_data
  ├── model (INDEX)
  ├── analysis_type
  ├── response
  ├── created_at (TIMESTAMP)
  ├── accessed_at (TIMESTAMP)
  └── hit_count
