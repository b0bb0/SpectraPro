1️⃣  DOCKER DEPLOYMENT (Recommended)
   
   Time: 3-5 minutes
   Complexity: Simple
   Best for: Production servers, cloud hosting, CI/CD
   
   Steps:
   $ cp .env.production .env
   $ nano .env          # Edit POSTGRES_PASSWORD, JWT_SECRET
   $ ./scripts/start-docker.sh
   
   Result: All services running in Docker containers
   Access: http://localhost (Caddy on port 80)

2️⃣  LOCAL DEPLOYMENT (MacBook Development)
   
   Time: 5-10 minutes
   Complexity: Moderate
   Best for: Development, quick testing
   Requirements: Node.js 20+, PostgreSQL 15+
   
   Steps:
   $ brew install postgresql@15  # (if not installed)
   $ cp .env.production .env
   $ nano .env          # Edit DATABASE_URL
   $ ./scripts/start-local.sh
   
   Result: Services running as native processes
   Access: http://localhost:3003 (frontend)
           http://localhost:5001 (API)
           http://localhost:5555 (Database GUI)
